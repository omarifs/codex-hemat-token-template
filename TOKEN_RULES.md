# TOKEN RULES

## 1) Selalu batasi ruang kerja
Contoh: "Ubah hanya file A dan B".

## 2) Beri data, bukan narasi
Masukkan log error singkat, bukan cerita panjang.

## 3) Pakai acceptance criteria numerik
Agen lebih cepat selesai jika target jelas.

## 4) Kurangi loop tanya-jawab
Tuliskan batasan sejak awal: dependency, style, test scope.

## 5) Pakai mode bertahap
- Step 1: Analisis singkat
- Step 2: Implementasi
- Step 3: Verifikasi

## 6) Simpan template tetap
System prompt yang stabil mencegah pengulangan token tiap sesi.

## 7) Minta format output tetap
Output terstruktur mempercepat evaluasi manusia dan agent.
