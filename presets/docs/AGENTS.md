# Documentation / Prompt Agent Rules

## Goal
Optimize for concise, reusable, and accurate documentation changes.

## Defaults
- Keep wording compact and specific.
- Prefer examples over long explanations.
- Reuse existing tone and structure.
- Avoid adding large prose blocks unless the user asked for them.

## Verification
- Check links, file references, and markdown structure.
- If the change affects instructions, ensure the example still matches the described workflow.
